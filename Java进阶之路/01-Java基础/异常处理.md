# [[异常处理]]

## 核心概念

### 异常体系

```
Throwable
├── Error（不可恢复）
│   ├── OutOfMemoryError
│   └── StackOverflowError
└── Exception
    ├── 受检异常（必须处理）
    │   ├── IOException
    │   └── SQLException
    └── 运行时异常（可选处理）
        ├── NullPointerException
        ├── ArrayIndexOutOfBoundsException
        └── IllegalArgumentException
```

### 异常处理机制

```java
// try-catch-finally
try { int r = 10 / 0; }
catch (ArithmeticException e) { e.printStackTrace(); }
finally { /* 一定执行 */ }

// try-with-resources（JDK7+ 自动关闭资源）
try (FileInputStream fis = new FileInputStream("test.txt")) {
    // 自动关闭
} catch (IOException e) { e.printStackTrace(); }
```

### 自定义异常

```java
public class BusinessException extends RuntimeException {
    private int code;
    public BusinessException(int code, String msg) {
        super(msg); this.code = code;
    }
}
```

## 踩坑记录

> [!warning] finally 中写 return 会覆盖异常
> ```java
> try { return compute(); }
> finally { return 0; }  // ❌ 永远返回 0
> ```

> [!warning] 空 catch 块（吞异常）
> ```java
> catch (Exception e) { }  // ❌ 异常没了，半夜线上出问题找不到原因
> ```

> [!warning] 不要用异常控制正常流程
> 创建异常对象需要填充栈轨迹，性能开销大。
