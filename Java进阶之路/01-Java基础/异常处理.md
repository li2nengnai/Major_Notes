# Java 异常处理

> 来源：[二哥的Java进阶之路](https://javabetter.cn/exception/gailan.html)

---

## 一、异常体系

```
Throwable
├── Error（严重错误，不可处理）
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── NoClassDefFoundError
│
└── Exception
    ├── 受检异常（Checked Exception，必须处理或抛出）
    │   ├── IOException
    │   ├── SQLException
    │   ├── ClassNotFoundException
    │   └── InterruptedException
    │
    └── 非受检异常（RuntimeException + Error，可不用处理）
        ├── NullPointerException
        ├── ArrayIndexOutOfBoundsException
        ├── IllegalArgumentException
        ├── ClassCastException
        └── ArithmeticException
```

## 二、异常处理机制

### try-catch-finally
```java
try {
    // 可能抛出异常的代码
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // 处理特定异常
    System.err.println("除数不能为0");
} catch (Exception e) {
    // 处理其他异常
    e.printStackTrace();
} finally {
    // 无论是否抛异常都会执行（通常用于释放资源）
    System.out.println("一定会执行");
}
```

### try-with-resources（JDK 7+）
```java
// 自动关闭实现了 AutoCloseable 的资源
try (FileInputStream fis = new FileInputStream("test.txt");
     BufferedReader br = new BufferedReader(new InputStreamReader(fis))) {
    
    String line = br.readLine();
    System.out.println(line);
} catch (IOException e) {
    e.printStackTrace();
}
// 不需要 finally 手动关闭资源
```

### throws 声明异常
```java
public void readFile(String path) throws IOException {
    // 方法不处理，交给调用者处理
    Files.readAllBytes(Paths.get(path));
}
```

### throw 手动抛异常
```java
public void checkAge(int age) {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException("年龄不合法: " + age);
    }
}
```

## 三、最佳实践

### ✅ 正确做法
```java
// 1. 精确捕获，不要吞异常
try { /* ... */ } catch (SpecificException e) {
    log.error("操作失败", e);
    throw e;  // 或包装后重新抛出
}

// 2. 使用自定义异常
public class BusinessException extends RuntimeException {
    private int code;
    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }
}

// 3. 资源使用 try-with-resources
try (Connection conn = getConnection()) { /* ... */ }

// 4. 记录异常日志
log.error("处理订单失败，orderId={}", orderId, e);
```

### ❌ 错误做法
```java
// 1. 捕获 Exception 基类（太宽泛）
catch (Exception e) { /* ... */ }

// 2. 空 catch 块（吞异常）
catch (Exception e) {}

// 3. 在 finally 中返回值（覆盖异常）
try { return compute(); } 
catch (Exception e) { return -1; } 
finally { return 0; }  // ❌ finally 返回覆盖 try 的返回值

// 4. 用异常控制流程（性能差）
try { /* ... */ } catch (Exception e) { /* 正常流程 */ }
```

## 四、异常性能开销

1. **创建异常对象**：需要填充栈轨迹（`fillInStackTrace()`），开销大
2. **JVM 优化**：无栈异常（`-XX:-OmitStackTraceInFastThrow`）
3. **不要用异常控制正常流程**

## 💡 面试高频题

1. **受检异常和非受检异常的区别？** → 受检必须处理/抛出，非受检可选
2. **final、finally、finalize 区别？** → 三个不同概念
3. **如果 try 中有 return，finally 还会执行吗？** → 会，finally 在 return 之前执行
4. **try-with-resources 的原理？** → 实现 AutoCloseable 接口，编译器自动生成 finally 关闭
5. **常见的 RuntimeException？** → NPE、IndexOutOfBounds、ClassCast、IllegalArgument
