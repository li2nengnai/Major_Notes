# 面向对象三大特性：封装、继承、多态

> 来源：[二哥的Java进阶之路](https://javabetter.cn/oo/encapsulation-inheritance-polymorphism.html)

---

## 一、封装（Encapsulation）

**核心思想**：将数据和操作数据的方法绑定在一起，对外隐藏内部实现细节

```java
public class Person {
    private String name;  // 私有字段
    private int age;

    // 公有 getter/setter 控制访问
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    
    public int getAge() { return age; }
    public void setAge(int age) {
        if (age > 0 && age < 150) {  // 校验逻辑
            this.age = age;
        }
    }
}
```

**优点**：
- 隐藏实现细节，提高安全性
- 降低耦合，便于修改维护
- 可对数据做校验和控制

**访问控制**：`private` → `default` → `protected` → `public`

## 二、继承（Inheritance）

**核心思想**：子类 `extends` 父类，复用父类属性和方法，并可以扩展

```java
public class Animal {
    protected String name;
    public void eat() { System.out.println("吃东西"); }
}

public class Dog extends Animal {
    public void bark() { System.out.println("汪汪"); }
    // 方法重写
    @Override
    public void eat() { System.out.println("啃骨头"); }
}
```

**关键规则**：
1. Java **单继承**，一个子类只能有一个直接父类（可通过接口实现多继承）
2. `super` 关键字调用父类构造方法/成员
3. 子类构造器必须调用父类构造器（默认调用 `super()`）
4. `final` 修饰的类不能被继承（如 `String`）
5. 不继承 `private` 成员和构造方法

**构造方法调用链**：
```
父类静态代码块 → 子类静态代码块 → 父类构造器 → 子类构造器
```

## 三、多态（Polymorphism）

**核心思想**：同一个行为具有多种不同表现形式

### 实现条件
1. 继承（或实现接口）
2. 方法重写（Override）
3. 父类引用指向子类对象

```java
Animal a = new Dog();  // 向上转型
a.eat();               // 输出：啃骨头（调用子类重写的方法）
// a.bark();           // ❌ 编译错误，父类引用不能调用子类特有方法
```

### 方法重写 vs 方法重载

| 对比 | 重写（Override） | 重载（Overload） |
|------|-----------------|-----------------|
| 范围 | 父子类之间 | 同一个类中 |
| 方法名 | 相同 | 相同 |
| 参数列表 | 相同 | 不同（数量/类型/顺序） |
| 返回值 | 相同或子类 | 无关 |
| 访问权限 | 不能更严格 | 无关 |
| 异常 | 不能抛出更宽的异常 | 无关 |

### 多态的好处
- 可扩展性：增加新子类不影响现有代码
- 接口统一：面向抽象/接口编程

---

## 四、抽象类与接口对比

| 对比项 | 抽象类 | 接口 |
|--------|--------|------|
| 关键字 | `abstract class` | `interface` |
| 继承方式 | `extends`（单继承） | `implements`（多实现） |
| 构造方法 | ✅ | ❌ |
| 成员变量 | 可以是任意类型 | `public static final` 常量 |
| 方法实现 | 可以有普通方法 | default/static 方法（Java 8+） |
| 设计思想 | is-a 关系 | like-a 能力/契约 |

---

## 💡 面试高频题

1. **Java 是单继承还是多继承？** → 单继承类、多实现接口
2. **构造方法能被重写吗？** → 不能，构造方法不是普通方法
3. **`@Override` 的作用？** → 编译器检查是否真的重写了父类方法
4. **向上转型和向下转型的区别？** → 向上自动、向下需要强制（`instanceof` 判断）
5. **静态方法能被重写吗？** → 不能，静态方法属于类，不存在多态
