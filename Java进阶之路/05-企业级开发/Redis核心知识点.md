# [[Redis]] 核心知识点

## 核心概念

### 五种数据结构

| 类型 | 场景 |
|------|------|
| String | 缓存、计数器、分布式锁 |
| List | 消息队列 |
| Set | 去重、共同好友 |
| ZSet | 排行榜 |
| Hash | 对象存储 |

### 持久化

| 对比 | RDB | AOF |
|------|-----|-----|
| 原理 | 快照全量数据 | 追加写命令 |
| 恢复速度 | 快 | 慢 |
| 数据安全 | 丢最近一次快照 | 按策略（always/everysec） |

### 缓存三大问题

| 问题 | 现象 | 解决 |
|------|------|------|
| **穿透** | 查不存在的数据打到 DB | 布隆过滤器/缓存空值 |
| **击穿** | 热点 key 过期 | 互斥锁/逻辑过期 |
| **雪崩** | 大量 key 同时过期 | 过期加随机值/多级缓存 |

### Redis 为什么快

**内存操作 + 单线程（避免上下文切换）+ IO 多路复用（epoll）**

## 踩坑记录

> [!warning] 缓存和数据库一致性
> **先更新数据库，再删除缓存**（Cache Aside Pattern）。不要先删缓存再更新数据库，否则并发时读到旧数据。

> [!warning] 分布式锁要设置过期时间
> 防止死锁。value 存线程唯一标识，Lua 脚本保证原子性。
