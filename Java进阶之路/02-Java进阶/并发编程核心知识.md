# [[并发编程]]核心知识

## 核心概念

### 线程 vs 进程

| 对比 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立内存 | 共享进程资源 |
| 切换开销 | 大 | 小 |
| 通信 | IPC | 共享内存 |

### 创建线程三种方式

```java
// 1. 继承 Thread
class MyThread extends Thread { public void run() { } }

// 2. 实现 Runnable（推荐）
class MyTask implements Runnable { public void run() { } }

// 3. Callable + FutureTask（带返回值）
FutureTask<Integer> ft = new FutureTask<>(() -> 42);
new Thread(ft).start();
Integer result = ft.get();
```

### synchronized 锁升级

```
无锁 → 偏向锁 → 轻量级锁（CAS） → 重量级锁（OS互斥量）
```

### volatile

- **可见性**：写立即刷新主存，读从主存读
- **禁止指令重排**：内存屏障
- **不能保证原子性**（如 `i++` 不是原子操作）

### 线程池核心参数

```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    corePoolSize,       // 核心线程数
    maximumPoolSize,    // 最大线程数
    keepAliveTime,      // 存活时间
    TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(queueSize),  // 任务队列
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.AbortPolicy()   // 拒绝策略
);
```

**工作流程**：核心线程→任务队列→最大线程→拒绝策略。

### 禁止 Executors 创建线程池

```java
Executors.newFixedThreadPool(10);   // ❌ 队列无界可能 OOM
Executors.newCachedThreadPool();    // ❌ 线程数无上限
```

## 踩坑记录

> [!warning] volatile 不保证原子性
> `count++` 是读-改-写三步操作，volatile 只能保证可见性，不能保证原子性。用 `AtomicInteger` 或 `synchronized`。

> [!warning] ThreadLocal 用完要 remove()
> ThreadLocal 的 key 是弱引用，value 是强引用。不 remove() 可能导致内存泄漏。
