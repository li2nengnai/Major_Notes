# 集合框架总览

> 来源：[二哥的Java进阶之路](https://javabetter.cn/collection/gailan.html)

---

## 一、集合体系结构

```
Collection（接口）
├── List（有序可重复）
│   ├── ArrayList     底层数组，查询快，增删慢
│   ├── LinkedList    底层双向链表，增删快，查询慢
│   └── Vector        线程安全（已过时）
│
├── Set（无序不可重复）
│   ├── HashSet       底层 HashMap
│   ├── LinkedHashSet 维护插入顺序
│   └── TreeSet       红黑树，可排序
│
└── Queue（队列）
    ├── PriorityQueue  优先级队列（堆）
    ├── ArrayDeque     双端队列
    └── LinkedList     也可作为队列

Map（接口）
├── HashMap         数组+链表+红黑树
├── LinkedHashMap   维护插入顺序
├── TreeMap         红黑树，key 可排序
├── Hashtable       线程安全（已过时）
└── ConcurrentHashMap 分段锁/CAS，线程安全
```

## 二、核心接口解析

### 2.1 Collection 接口
```java
boolean add(E e);
boolean remove(Object o);
int size();
boolean isEmpty();

```

### 2.2 List vs Set vs Queue
| 对比 | List | Set | Queue |
|------|------|-----|-------|
| 顺序 | 有序 | 无序(L/H有序) | FIFO |
| 重复 | ✅ | ❌ | ✅ |
| 随机访问 | ✅（索引） | ❌ | ❌ |
| null | ✅ | 1个 | 视实现 |

### 2.3 Map 接口
```java
V put(K key, V value);
V get(Object key);
V remove(Object key);
boolean containsKey(Object key);
Set<Map.Entry<K,V>> entrySet();
Set<K> keySet();
```

## 三、快速选型指南

| 场景 | 推荐实现 |
|------|---------|
| 随机访问、尾部追加 | `ArrayList` |
| 频繁增删中间元素 | `LinkedList` |
| 键值对存储 | `HashMap` |
| 线程安全 Map | `ConcurrentHashMap` |
| 需要有序键值对 | `LinkedHashMap` |
| Key 需要排序 | `TreeMap` |
| 去重 | `HashSet` |
| 去重+有序 | `LinkedHashSet` |
| 去重+排序 | `TreeSet` |
| 消息队列 | `ArrayDeque` |
| 优先级任务 | `PriorityQueue` |

## 四、ArrayList 核心要点

```java
// 继承关系
ArrayList → AbstractList → Collection → Iterable

// 核心参数
private static final int DEFAULT_CAPACITY = 10;
transient Object[] elementData;
private int size;

// 扩容机制
int newCapacity = oldCapacity + (oldCapacity >> 1);  // 1.5 倍
// Arrays.copyOf() 拷贝到新数组
```

- 底层用 `Object[]` 存储，默认容量 10
- 扩容 1.5 倍 → `int newCapacity = oldCapacity + (oldCapacity >> 1)`
- 扩容时 `Arrays.copyOf()` 拷贝，效率低
- 批量插入时 `ensureCapacityInternal()` 手动指定容量可提升性能

## 五、LinkedList 核心要点

```java
LinkedList → AbstractSequentialList → Collection → Iterable
           → Deque → Queue

// 内部节点
private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;
}
```

- 双向链表结构，不扩容
- 随机访问 O(n)，头尾操作 O(1)
- 可作为 List、Queue、Deque 使用
- 内存占用比 ArrayList 大（存储前后指针）

## 六、快速对比：ArrayList vs LinkedList

| 对比 | ArrayList | LinkedList |
|------|-----------|------------|
| 底层结构 | 数组 | 双向链表 |
| 随机访问 | O(1) | O(n) |
| 头部插入 | O(n) | O(1) |
| 尾部插入 | O(1) | O(1) |
| 中间插入 | O(n) | O(n) |
| 内存占用 | 少 | 多（前后指针） |
| 扩容开销 | 有 | 无 |
