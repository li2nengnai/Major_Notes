# [[面向对象]]（OOP）基础

## 核心概念

### 类与对象

- **类** = 模板/蓝图（图纸）
- **对象** = 根据类创建的具体实例（盖好的房子）

```java
// 类 = 图纸
public class Student {
    String name;            // 属性
    public void study() { } // 行为
}

// 对象 = 具体的房子
Student s1 = new Student();    // new 在堆内存开空间
s1.name = "播妞";
s1.study();
```

### [[封装]]（`private`）

用 `private` 隐藏成员变量，通过 `getter/setter` 暴露访问接口。好处是可以在 setter 中加校验逻辑。

```java
public void setAge(int age) {
    if (age > 0 && age < 150) {  // 校验
        this.age = age;
    }
}
```

### [[构造器]]

创建对象时自动调用的特殊方法，方法名与类名相同，**无返回值**。

- 没写任何构造器 → 系统提供默认无参构造
- **写了有参构造后，必须手动写无参构造**

### `this` 关键字

代表当前对象的引用，用来区分成员变量和参数：

```java
public void setScore(double score) {
    this.score = score;  // 左边的 this.score 是成员变量
}
```

### [[JavaBean]] 标准

私有成员变量 + 无参构造 + getter/setter

## 语法格式

### 类定义

```java
public class 类名 {
    // 成员变量（属性）
    数据类型 变量名;

    // 成员方法（行为）
    public 返回值 方法名() { }
}
```

### 创建对象

```java
类名 对象名 = new 类名();
对象名.成员变量 = 值;
对象名.成员方法();
```

## 踩坑记录

> [!warning] 写了有参构造后无参构造消失
> **现象**：`new Student()` 报编译错误
> ```java
> public Student(String name) { }  // 写了有参
> // 无参构造被覆盖了！
> Student s = new Student();  // ❌
> ```
> **解决**：手动写上无参构造 `public Student() {}`

> [!warning] this 不能在静态方法中使用
> ```java
> public static void test() {
>     System.out.println(this.name);  // ❌ 静态方法没有 this
> }
> ```

## 实战案例

```java
// JavaBean 标准写法
public class Movie {
    private int id;
    private String name;

    public Movie() { }
    public Movie(int id, String name) {
        this.id = id;
        this.name = name;
    }
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    // ... 其他字段同理
}
```
