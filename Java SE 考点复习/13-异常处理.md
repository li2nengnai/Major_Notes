## 核心概念

### [[Java 异常体系]]

```
Throwable
├── Error（严重错误，不可处理）
│   ├── OutOfMemoryError
│   └── StackOverflowError
└── Exception
    ├── 运行时异常（RuntimeException）→ 可不处理
    │   ├── NullPointerException
    │   ├── ArrayIndexOutOfBoundsException
    │   └── ArithmeticException
    └── 编译时异常（Checked Exception）→ 必须处理
        ├── IOException
        ├── SQLException
        └── ParseException
```

**关键区别**：编译时异常不处理→编译不通过；运行时异常不处理→编译通过，运行可能崩溃。

### `throws` vs `throw`

| 关键字 | 位置 | 作用 | 后跟 |
|--------|------|------|------|
| `throws` | 方法声明上 | 声明可能抛出的异常 | 异常类型 |
| `throw` | 方法体内 | 手动抛出异常对象 | 异常对象 |

### `finally`

无论是否异常都会执行，在 `return` 之前执行。**唯一阻止 `finally` 的方式：`System.exit(0)`**

## 语法格式

```java
try {
    // 可能异常的代码
} catch (具体异常类型 e) {
    // 处理异常
} finally {
    // 必须执行的代码
}

// try-with-resources（JDK7+，自动关闭资源）
try (FileInputStream fis = new FileInputStream("a.txt")) {
    // 用 fis，自动关闭
} catch (IOException e) {
    e.printStackTrace();
}
```

## 踩坑记录

> [!warning] finally 中不要写 return
> **现象**：try 中 return 1，结果返回了 2
> ```java
> public static int test() {
>     try { return 1; }
>     finally { return 2; }  // ❌ 覆盖了 1
> }
> // test() → 2
> ```
> **原则**：finally 只用来释放资源，不返回数据。

> [!warning] 空 catch 块（吞异常）
> ```java
> try { /* 可能异常 */
> } catch (Exception e) { }  // ❌ 异常被吞了，出问题不知道原因
> ```
> 至少打日志：`catch (Exception e) { e.printStackTrace(); }`

## 实战案例

```java
// 自定义异常
public class AgeIllegalException extends Exception {
    public AgeIllegalException(String message) { super(message); }
}

public void saveAge(int age) throws AgeIllegalException {
    if (age < 0 || age > 150) {
        throw new AgeIllegalException("年龄不合法：" + age);
    }
}
```
