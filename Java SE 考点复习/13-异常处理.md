# 异常处理

## 📌 课程模块

pk014 · 异常体系/分类/try-catch/throws/自定义异常 · **理解考点**

## 💻 核心代码示例

```java
// 1. 编译时异常必须处理（两种方式：throws 或 try-catch）
// 方式1：throws 声明抛出
public class ExceptionTest1 {
    public static void main(String[] args) throws Exception {
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        Date d = sdf.parse("2028-11-11 10:24");   // ❗ 编译时异常，必须处理
    }
}

// 方式2：try-catch 捕获处理
public class ExceptionTest1 {
    public static void main(String[] args) {
        try {
            SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
            Date d = sdf.parse("2028-11-11 10:24");
        } catch (ParseException e) {
            e.printStackTrace();               // 打印异常信息
        }
    }
}
```

```java
// 2. 自定义异常 + throw + throws
// 自定义编译时异常
public class AgeIllegalException extends Exception {
    public AgeIllegalException(String message) {
        super(message);
    }
}

// 自定义运行时异常
public class AgeIllegalRuntimeException extends RuntimeException {
    public AgeIllegalRuntimeException(String message) {
        super(message);
    }
}

// 使用异常
public class ExceptionTest2 {
    public static void main(String[] args) {
        try {
            saveAge2(225);                     // 编译时异常必须try-catch
        } catch (AgeIllegalException e) {
            e.printStackTrace();
        }

        saveAge(225);                          // 运行时异常可不处理（但建议处理）
    }

    // throws 声明抛出编译时异常
    public static void saveAge2(int age) throws AgeIllegalException {
        if (age <= 0 || age >= 150) {
            throw new AgeIllegalException("年龄不合法：" + age);
        }
        System.out.println("年龄保存成功：" + age);
    }

    // 运行时异常（无需throws声明）
    public static void saveAge(int age) {
        if (age <= 0 || age >= 150) {
            throw new AgeIllegalRuntimeException("年龄不合法：" + age);
        }
        System.out.println("年龄保存成功：" + age);
    }
}
```

```java
// 3. try-catch-finally（finally一定会执行）
public class Test1 {
    public static void main(String[] args) {
        try {
            System.out.println(10 / 2);
            // return;                   // finally 在 return 之前执行
            // System.exit(0);           // ❗ 只有这个能阻止 finally 执行
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("===finally执行了一次===");  // 无论是否异常都执行
        }
    }
}
```

```java
// 4. finally 中不要写 return！（会覆盖 try/catch 中的返回值）
public static int chu(int a, int b) {
    try {
        return a / b;                    // 正常返回结果
    } catch (Exception e) {
        e.printStackTrace();
        return -1;                       // 异常返回 -1
    } finally {
        // ❌ 千万不要在finally中返回数据！会覆盖try中的return
        return 111;                      // 永远返回111，覆盖了上面的return
    }
}
// chu(10, 2) → 返回 111（不是5！）
// chu(10, 0) → 返回 111（不是-1！）
```

## 🧩 知识点拆解

### 知识点1：异常体系

- **是什么**：Java 异常顶层是 `Throwable`，分为 `Error`（错误）和 `Exception`（异常）
- **Exception 分类**：
  - **编译时异常**（受检异常）：必须处理（如 `ParseException`）
  - **运行时异常**（非受检异常）：可以不处理（如 `NullPointerException`、`ArrayIndexOutOfBoundsException`）
- **老师强调的要点**：
  - 编译时异常不处理 → **编译不通过**
  - 运行时异常不处理 → 编译通过，运行时可能崩溃

### 知识点2：throw vs throws

- **是什么**：
  - `throw`：方法体内，手动**抛出**异常对象
  - `throws`：方法声明上，**声明**方法可能抛出的异常类型
- **老师强调的要点**（**必考！**）：
  - `throw` 后跟异常对象（`throw new XxxException()`）
  - `throws` 后跟异常类型（`throws XxxException`）
  - `throw` 在方法内部，`throws` 在方法签名上

### 知识点3：finally

- **是什么**：无论是否发生异常，`finally` 中的代码都会执行
- **老师强调的要点**（**易错点！**）：
  - `finally` 在 `return` 之前执行（除非 `System.exit(0)`）
  - **千万不要在 `finally` 中写 `return`**，会覆盖 try/catch 中的返回值

## ⚠️ 常见考题 / 易错点

1. **选择题——finally 执行时机**：`try { return 1; } finally { return 2; }` → 返回 **2**

2. **选择题——throws vs throw**：以下说法正确的是？
   - `throws` 在方法声明上 ✅
   - `throw` 在方法体内 ✅
   - `throws` 后跟异常对象 ❌（跟异常类型）

3. **选择题——运行时异常 vs 编译时异常**：
   - `RuntimeException` → 运行时异常 ✅
   - `IOException` → 编译时异常 ✅
   - `NullPointerException` → 运行时异常 ✅

4. **改错题——自定义异常**：
   ```java
   // 编译时异常应继承 Exception
   public class MyException extends Exception { }
   // 运行时异常应继承 RuntimeException
   public class MyRuntimeException extends RuntimeException { }
   ```

5. **编程题——异常处理**：在可能出现异常的地方适当使用 try-catch 或 throws
