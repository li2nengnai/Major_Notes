# [[类型转换]]

## 📌 课程模块

pk002 · 自动类型转换与强制类型转换 · **高频考点** · **易错考点**

## 核心概念

### 自动类型转换

小范围类型**自动**变为大范围类型，无需手动操作：

```
byte → short → int → long → float → double
               ↑
             char
```

- `char` 可以自动转 `int`（本质是 ASCII/Unicode 数字）
- 表达式中多种类型时，结果自动升为最大类型

### 强制类型转换

大范围类型手动转为小范围类型，语法 `(目标类型) 数据`：

```java
double d = 99.5;
int m = (int) d;   // 99（截断小数，不是四舍五入！）
```

## 语法格式

```java
// 自动转换
byte a = 12;
int b = a;          // byte → int，自动

// 强制转换
int i = 1500;
byte j = (byte) i;  // 数据溢出！byte 范围 -128~127
```

## 踩坑记录

> [!warning] 强制转换丢失精度
> **现象**：`1500` 强转 byte 后值异常。
> **原因**：byte 范围 -128~127，1500 超出范围，高位截断后值不对。
> ```java
> int i = 1500;          // 二进制：00000000 00000000 00000101 11011100
> byte j = (byte) i;     // 截断后：11011100 → -36（补码）
> ```
> **解决**：确保值在目标类型范围内。

> [!warning] 小数强转整数不会四舍五入
> ```java
> double d = 99.99;
> int i = (int) d;       // 99，不是 100！
> ```

## 实战案例

```java
// 常见的类型提升陷阱
int a = 5;
double b = 2;
System.out.println(a / b);  // 2.5（int + double → double）
System.out.println(a / 2);  // 2（int / int → int，取整）
```
