## 核心概念

### [[Socket]] 通信模型

```
服务端：ServerSocket(port) → accept() → 等待客户端连接
客户端：Socket(host, port) → 连接服务端
连接后：InputStream/OutputStream 收发数据
```

### [[ServerSocket]]

```java
ServerSocket server = new ServerSocket(8888);
Socket client = server.accept();  // 阻塞等待
```

### [[Socket]]

```java
Socket socket = new Socket("localhost", 8888);
```

**多线程处理双向通信**：收发分别用独立线程，避免相互阻塞。

## 语法格式

```java
// 服务端
ServerSocket serverSocket = new ServerSocket(8888);
Socket clientSocket = serverSocket.accept();

// 接收消息
BufferedReader reader = new BufferedReader(
    new InputStreamReader(clientSocket.getInputStream()));
String msg = reader.readLine();

// 发送消息
PrintWriter writer = new PrintWriter(clientSocket.getOutputStream(), true);
writer.println("消息内容");
```

## 踩坑记录

> [!warning] Socket 通信阻塞
> **现象**：程序卡住不动
> **原因**：`accept()` / `readLine()` 都是阻塞方法，没有客户端连接或没有数据时会一直等。
> **解决**：用多线程分别处理收发，或用 `setSoTimeout()` 设置超时。

> [!warning] Socket 用完要关闭
> 不关会导致端口被占用。用 try-with-resources 自动关闭。
