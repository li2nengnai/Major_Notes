# IO 流（字节流 / 字符流 / 缓冲流 / 对象流）

## 💻 核心代码示例

```java
// 1. File 对象（文件/目录）
public class FileTest1 {
    public static void main(String[] args) {
        // 绝对路径
        File f1 = new File("D:/resource/ab.txt");
        System.out.println(f1.length());       // 文件大小（字节）
        System.out.println(f1.exists());       // 是否存在

        // File 可以指代不存在的路径
        File f3 = new File("D:/resource/aaaa.txt");
        System.out.println(f3.exists());       // false（不报错）

        // 相对路径（从工程根目录开始）
        File f4 = new File("file-io-app\\src\\itheima.txt");
        System.out.println(f4.length());
    }
}
```

```java
// 2. 字节流 — 一次读取一个字节（性能差，汉字乱码）
public class FileInputStreamTest1 {
    public static void main(String[] args) throws Exception {
        InputStream is = new FileInputStream("file-io-app\\src\\itheima01.txt");

        int b;                               // 用于记住读取的字节
        while ((b = is.read()) != -1) {      // read()返回-1表示读完
            System.out.print((char) b);      // 汉字会乱码！一个汉字多个字节
        }

        // ❗ 读取数据的性能很差！
        // ❗ 读取汉字输出会乱码（一个汉字占3个字节）
        // ❗ 流使用完毕之后，必须关闭！释放系统资源！
        is.close();
    }
}
```

```java
// 3. 字节流 — 一次读完所有字节（推荐小文件）
public class FileInputStreamTest3 {
    public static void main(String[] args) throws Exception {
        InputStream is = new FileInputStream("file-io-app\\src\\itheima03.txt");

        // JDK9+ 方式：readAllBytes() 一次读完
        byte[] buffer = is.readAllBytes();
        System.out.println(new String(buffer));

        is.close();
    }
}
```

```java
// 4. 缓冲流（提高性能，推荐方式）
public class BufferedInputStreamTest1 {
    public static void main(String[] args) {
        try (
                InputStream is = new FileInputStream("src/itheima01.txt");
                InputStream bis = new BufferedInputStream(is);       // 包装
                OutputStream os = new FileOutputStream("dest.txt");
                OutputStream bos = new BufferedOutputStream(os);     // 包装
        ) {
            byte[] buffer = new byte[1024];
            int len;
            while ((len = bis.read(buffer)) != -1) {
                bos.write(buffer, 0, len);
            }
            System.out.println("复制完成！！");
        } catch (Exception e) {
            e.printStackTrace();
        }
        // ❗ try-with-resources 自动关闭资源，无需手动 close
    }
}
```

```java
// 5. 字符流（适合文本文件，无乱码）
public class FileReaderTest1 {
    public static void main(String[] args) {
        try (Reader fr = new FileReader("io-app2\\src\\itheima01.txt")) {
            char[] buffer = new char[3];           // 字符数组，不是字节数组
            int len;                               // 记录每次读取的字符数
            while ((len = fr.read(buffer)) != -1) {
                // 读取多少倒出多少，避免读到上次残留
                System.out.print(new String(buffer, 0, len));
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

```java
// 6. 对象流 — 序列化/反序列化
// 序列化（写对象）
public class Test1ObjectOutputStream {
    public static void main(String[] args) {
        try (ObjectOutputStream oos =
                     new ObjectOutputStream(new FileOutputStream("data.txt"))) {
            User u = new User("admin", "张三", 32, "666888xyz");
            oos.writeObject(u);                    // ❗ 类必须实现 Serializable
            System.out.println("序列化成功！");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

// 反序列化（读对象）
public class Test2ObjectInputStream {
    public static void main(String[] args) {
        try (ObjectInputStream ois =
                     new ObjectInputStream(new FileInputStream("data.txt"))) {
            User u = (User) ois.readObject();      // 返回 Object，需强制转换
            System.out.println(u);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

## 🧩 知识点拆解

### 知识点1：字节流 vs 字符流

- **是什么**：
  - 字节流：`InputStream`/`OutputStream`，操作所有文件（图片、视频、文本等）
  - 字符流：`Reader`/`Writer`，操作纯文本文件（无乱码问题）
- **老师强调的要点**：
  - 字节流读取汉字可能乱码（一个汉字 3 个字节，一次读一个字节会拆散）
  - 字符流按字符读取，读取文本文件无乱码
  - 拷贝文件用字节流（通用），读取文本用字符流

### 知识点2：缓冲流

- **是什么**：`BufferedInputStream`/`BufferedOutputStream`，包装基础流，提高效率
- **老师强调的要点**：
  - 缓冲流自带 8KB 缓冲区，减少磁盘 I/O 次数
  - 用**装饰器模式**包装基础流：`new BufferedInputStream(new FileInputStream(...))`
  - 推荐配合 `byte[]` 或 `char[]` 数组使用，效率更高

### 知识点3：try-with-resources

- **是什么**：JDK7 引入的自动资源管理语法
- **语法**：`try (资源声明) { 使用资源 } catch { 处理异常 }`
- **老师强调的要点**：
  - 资源必须实现 `AutoCloseable` 接口
  - 多个资源用 `;` 分隔
  - **资源会自动关闭**，无需在 finally 中手动 close

### 知识点4：对象序列化

- **是什么**：将 Java 对象转换为字节序列保存到文件或网络传输
- **老师强调的要点**：
  - 类必须实现 `Serializable` 接口（标记接口，无抽象方法）
  - `transient` 修饰的成员变量不会被序列化
  - 反序列化时需强制类型转换

## ⚠️ 常见考题 / 易错点

1. **选择题——字节流 vs 字符流**：以下哪个适合复制图片？
   - `FileReader` ❌（字符流处理图片会损坏）
   - `FileInputStream` ✅（字节流，适合所有类型文件）

2. **填空题——序列化要求**：类必须实现 `Serializable` 接口

3. **改错题——资源关闭**：
   ```java
   // ❌ 老式写法，finally中容易漏掉close
   InputStream is = new FileInputStream("a.txt");
   // 使用...
   is.close();
   
   // ✅ 推荐：try-with-resources
   try (InputStream is = new FileInputStream("a.txt")) {
       // 使用... 自动关闭
   }
   ```

4. **选择题——缓冲流作用**：提高读写效率，减少磁盘 I/O 次数

5. **填空题——transient 关键字**：修饰的成员变量**不参与序列化**

6. **编程题——文件复制**：缓冲流 + byte[] 数组，边读边写
