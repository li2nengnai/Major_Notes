# [[继承]]、[[多态]]、[[抽象类]]、[[接口]]

## 📌 课程模块

pk009-pk010 · 继承/方法重写/多态/final/抽象类/接口 · **高频考点**

## 核心概念

### [[继承]]（extends）

子类继承父类，获得父类的非私有成员。Java **单继承**（一个类只能有一个直接父类）。

```java
class B extends A { }  // B 拥有 A 的非私有成员
```

### [[方法重写]]（@Override）

子类改写父类方法，方法签名必须相同。`private` 和 `final` 方法不能被重写。

### [[多态]]

**一个对象多种形态**：`父类引用 = new 子类对象()`

- **行为看子类**：`p1.run()` 调子类重写的方法
- **变量看父类**：`p1.name` 访问父类的变量

```java
People p = new Teacher();
p.run();          // Teacher 的 run
System.out.println(p.name);  // People 的 name
```

### [[抽象类]] vs [[接口]]

| 对比 | 抽象类 | 接口 |
|------|--------|------|
| 关键字 | `abstract class` | `interface` |
| 构造器 | 可以有 | ❌ |
| 成员变量 | 任意类型 | `public static final` 常量 |
| 方法 | 抽象+具体 | JDK8+ 可 default/static |
| 设计思想 | is-a 关系 | like-a 能力/契约 |

## 语法格式

```java
// 抽象类 + 模板方法
public abstract class People {
    public final void write() {           // 模板方法（不让子类改流程）
        System.out.println(writeMain());  // 子类实现内容
    }
    public abstract String writeMain();   // 子类必须实现
}

// 接口
public interface A {
    String NAME = "常量";  // public static final
    void test();           // public abstract
}
```

## 踩坑记录

> [!warning] 多态中不能调子类特有方法
> ```java
> Animal a = new Dog();
> a.eat();          // ✅ 父类有的方法
> a.watchDoor();    // ❌ 编译错误！父类没有 watchDoor
> ```
> **解决**：向下转型 `((Dog) a).watchDoor();` 前用 `instanceof` 判断类型。

> [!warning] 静态方法不能被重写
> 静态方法属于类，子类写同名静态方法叫「方法隐藏」，不是重写。运行时调哪个看引用类型，不看实际对象。

## 实战案例

```java
// 模板方法设计模式：作文模板
Student s = new Student();
s.write();  // 自动执行模板，正文内容由子类实现
```
