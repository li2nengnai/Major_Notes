# 继承、多态、抽象类、接口

## 📌 课程模块

pk009-pk010 · 继承/方法重写/多态/final/抽象类/接口 · **高频考点**

## 💻 核心代码示例

```java
// 1. 继承 extends
class B extends A { }          // B 继承 A，B 拥有 A 的非私有成员

// 继承的特点
// B b = new B();
// System.out.println(b.i);        // 可以访问 public 成员
// System.out.println(b.j);        // 不可以访问 private 成员
// b.print1();                     // 可以调用 public 方法
// b.print2();                     // 不可以调用 private 方法
```

```java
// 2. 方法重写 @Override
public class Student {
    @Override
    public String toString() {           // 重写 Object 的 toString
        return "Student{name=" + name + "}";
    }
}

public class Test {
    public static void main(String[] args) {
        Student s = new Student("播妞", 19);
        System.out.println(s.toString());  // 调用重写后的 toString
        System.out.println(s);             // 同上（println 自动调 toString）
    }
}
```

```java
// 3. 多态：对象多态 + 行为多态
public class Test {
    public static void main(String[] args) {
        // 对象多态：父类引用指向子类对象
        People p1 = new Teacher();
        People p2 = new Student();

        // ⭐ 行为多态：编译看左边（父类），运行看右边（子类方法）
        p1.run();                        // 调用的是 Teacher 的 run()
        p2.run();                        // 调用的是 Student 的 run()

        // ⭐ 变量：编译看左边，运行看左边（访问父类变量）
        System.out.println(p1.name);     // 访问的是 People 的 name
    }
}
```

```java
// 4. 抽象类 abstract + 模板方法设计模式
public abstract class People {
    // 模板方法（final修饰，子类不能重写）
    public final void write() {
        System.out.println("《我的爸爸》");
        System.out.println(writeMain()); // 调用抽象方法（子类实现）
        System.out.println("结尾");
    }

    // 抽象方法（没有方法体，子类必须实现）
    public abstract String writeMain();
}

public class Student extends People {
    @Override
    public String writeMain() {
        return "我的爸爸是超人...";
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        Student s = new Student();
        s.write();  // 调用模板方法，自动执行子类实现的正文
    }
}
```

```java
// 5. 接口 interface
public interface A {
    String SCHOOL_NAME = "极客程序员";    // 常量（public static final）
    void test();                          // 抽象方法（public abstract）
}

// 实现类（implements 可以同时实现多个接口）
public class D implements A {
    @Override
    public void test() {
        System.out.println("实现方法");
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        System.out.println(A.SCHOOL_NAME);  // 接口名.常量名访问
        // A a = new A();  // ❌ 接口不能 new
        D d = new D();
        d.test();
    }
}
```

## 🧩 知识点拆解

### 知识点1：继承

- **是什么**：`class B extends A`，B 继承 A，B 获得 A 的非私有成员
- **老师强调的要点**：
  - Java 是**单继承**（一个类只能有一个直接父类）
  - 子类不能继承父类的 `private` 成员
  - 子类可以**重写**父类的方法（方法签名必须相同）
  - 继承体现 is-a 关系（学生 is-a 人）

### 知识点2：方法重写（Override）

- **是什么**：子类重写父类的方法，方法签名（方法名+参数列表）完全相同
- **在代码中的作用**：子类改写 Object 的 `toString()` 方法，返回有意义的字符串
- **老师强调的要点**（**高频考点！**）：
  - `@Override` 注解：检查是否重写正确（建议都加上）
  - 重写返回值类型**可以相同或为原返回类型的子类型**
  - `private` 方法不能被重写
  - `final` 方法不能被重写
  - **重写 vs 重载**：重写是父子类之间，重载是同类的不同参数

### 知识点3：多态（重要！）

- **是什么**：同一个类型在不同时刻表现出不同形态
- **两个前提**：继承关系 + 方法重写
- **老师强调的要点**（**必考题！**）：
  - **行为看子类**：`p1.run()` 调用子类重写的方法
  - **变量看父类**：`p1.name` 访问父类的变量
  - 多态的优势：解耦，提高扩展性
  - 多态的劣势：不能调用子类特有方法（需向下转型 `(子类类型) 父类引用`）

### 知识点4：抽象类 vs 接口

- **是什么**：两者都不能 new，都需要子类/实现类
- **对比**：
  | 比较项 | 抽象类 | 接口 |
  |--------|--------|------|
  | 关键字 | `abstract class` | `interface` |
  | 构造器 | 可以有 | 不能有 |
  | 成员变量 | 可有各种类型 | 只能 public static final 常量 |
  | 方法 | 抽象/具体方法均可 | JDK8前只能抽象方法 |
  | 继承/实现 | extends（单继承） | implements（多实现） |
- **老师强调的要点**：
  - **选择**：有 is-a 关系用抽象类，有 like-a 能力用接口
  - **模板方法模式**：抽象类定义算法骨架（`final`），子类实现可变部分

## ⚠️ 常见考题 / 易错点

1. **选择题——多态调用结果**：
   ```java
   Animal a = new Dog();
   a.eat();    // 调用 Dog 的 eat（运行看右边）
   // a.watchDoor(); // ❌ 编译报错（编译看左边，父类没有此方法）
   ```

2. **选择题——重写 vs 重载**：以下说法正确的是？
   - 重写是父子类之间的，方法签名相同 ✅
   - 重载是同类中的，方法名相同参数不同 ✅
   - 重写可以改变方法名 ❌

3. **改错题——抽象类实例化**：`new A()` 其中 A 是抽象类 → **编译错误**

4. **填空题——接口特点**：接口中的变量默认是 `public static final`，方法默认是 `public abstract`

5. **选择题——模板方法模式**：模板方法用哪个关键字修饰防止被子类重写？→ **final**
