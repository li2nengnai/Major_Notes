# [[方法]]定义与重载

## 📌 课程模块

pk005 · 方法定义/参数传递/返回值/重载 · **高频考点**

## 核心概念

### 方法定义

```java
修饰符 返回值类型 方法名(参数列表) { 方法体; return 值; }
```

- 有返回值必须写 `return 值;`
- 无返回值用 `void`，可写 `return;` 提前结束方法

### [[值传递]]

Java **都是值传递**——基本类型传值的副本，方法内修改不影响原变量：

```java
int a = 10;
change(a);
System.out.println(a);  // 10（没变！）

void change(int x) {    // x 是 a 的副本
    x = 20;             // 改的是副本
}
```

### [[方法重载]]（Overload）

同一类中，方法名相同，参数列表不同（类型/个数/顺序），**与返回值无关**。

```java
void test() { }
void test(int a) { }          // ✅ 参数个数不同
void test(double a) { }       // ✅ 参数类型不同
int test(int a, int b) { }    // ✅
// int test(int x, int y) { } // ❌ 仅参数名不同，不构成重载
```

## 语法格式

```java
public static int sum(int a, int b) {
    return a + b;
}

// 调用
int result = sum(10, 20);
```

## 实战案例

```java
// return 提前结束方法（用于校验）
public static void chu(int a, int b) {
    if (b == 0) {
        System.out.println("不能除0");
        return;                // 结束方法
    }
    int c = a / b;
    System.out.println("结果是：" + c);
}
```

## 踩坑记录

> [!warning] 仅返回值不同不构成重载
> ```java
> void test(int a) { }
> int test(int a) { return 1; }  // ❌ 编译错误：方法已存在
> ```
> 编译器只看**方法名+参数列表**，不看返回值。

> [!warning] 参数名不同不构成重载
> ```java
> void test(int a) { }
> void test(int b) { }  // ❌ 编译错误：参数名不同不算
> ```
